# Git Setup & Document Your Knowledge

## Research Git basics and write your reflections in git_knowledge.md:

### Have you used Git before? If so, in what context?

Yes, I have used Git before mainly for managing code and tracking changes in my projects. During my university assignments and group projects, Git helped me store my code safely and keep track of different versions of my work. It was also useful when working with teammates because everyone could contribute to the same repository without overwriting each other's work.

### Which Git client (if any) did you choose? Why?

I chose to use Git through VS Code along with the Git command line. Since I already use VS Code for coding, it makes the workflow easier because I can manage my code, see file changes, commit updates, and push them to GitHub in the same place. The built-in Git tools in VS Code are simple to use and help visualize changes clearly.

### What was the most interesting thing you learned about Git today?

One of the most interesting things I learned is how Git keeps track of the complete history of changes in a project. Every commit acts like a checkpoint, which means I can go back to a previous version of the code if something breaks. This makes development much safer and more organized, especially when experimenting with new ideas.
